# Tandem Robotics · GitHub PBL Docs Hub

Official documentation hub for Tandem Robotics GitHub PBL coaching and learning paths.

이 레포는 **Tandem Robotics GitHub PBL 코칭 서비스의 유일한 안내판**입니다.  
처음 오셨다면, 아래 순서만 따라오시면 됩니다.

---

## 🟨 PBL Starter (가장 먼저 시작하세요)

GitHub PBL은  
코드를 “따라 치는 과정”이 아니라  
**문제를 정의하고, 기록하고, 설계하는 경험**을 목표로 합니다.

시작하기 전에, **딱 한 가지만 확인하세요.**

---

### 🔍 시작 전 체크

아래 질문에 **“아직 헷갈린다”**가 하나라도 있다면  
👉 **SelfAbot 사전 단계**를 먼저 권장합니다.

- Arduino 코드에서 `setup()` / `loop()`는 알겠는데  
  **코드가 왜 이렇게 나뉘는지는 잘 모르겠다**
- C 문법은 봤지만  
  **C++ 클래스 / 객체 구조는 아직 감이 없다**
- 로봇 코드가  
  **“동작은 되는데, 왜 이렇게 설계됐는지” 설명하기 어렵다**

그렇지 않다면  
👉 **바로 Tandem Robotics PBL로 진행해도 괜찮습니다.**

---

### 🔹 선택 1 — SelfAbot (사전 가속 단계 · 선택)

**SelfAbot은 기초 문법 강의가 아닙니다.**

SelfAbot은  
- C 문법에 익숙한 초보자가  
- **C++ 코드 구조를 ‘로봇 동작의 역할 분리’ 관점에서 이해하도록 돕는**  
- Tandem Robotics PBL을 위한 **선택형 사전 가속 단계**입니다.

이 단계에서는  
- 클래스 = “로봇의 부품 역할”  
- 함수 분리 = “동작 책임 분리”  
- C → C++ 전환을 **개념 설명이 아닌 실제 동작 코드로** 경험합니다.

👉 **[SelfAbot 저장소](https://github.com/wookjin-chung/SelfAbot)**  
※ 이 단계는 **필수가 아닙니다.**

---

### 🔹 선택 2 — Tandem Robotics PBL (바로 시작)

이미  
- 코드 구조를 읽는 데 큰 부담이 없거나  
- Python ↔ Arduino 구조에 바로 도전하고 싶다면  

👉 **아래 PBL Starter 템플릿으로 바로 진행하세요.**

👉 **[PBL Starter Template](https://github.com/wookjin-chung/tandem-pbl-starter-template)**

---

### 🚀 Quick Start (요약)

1. PBL Starter Template을 **Fork**
2. 문제 정의 → 실험 → 기록을 **GitHub에 남기기**
3. 결과보다 **사고 과정과 결정 이유**를 기록
4. (선택) AI 코칭을 활용해 설계 확장

## 🟦 Legacy v1 (현재 공개된 실습 구조)

이미 공개된 **Tandem Robotics v1 구조**를 기반으로  
- micro:bit + Arduino(UNO)  
- Python 중심 실습  
- I2C 통신 구조를 경험할 수 있습니다.

이 경로는  
- 기존 자료를 활용해 빠르게 실습하고 싶은 분  
- 현재 배포 중인 교재/자료와 동일한 구조가 필요한 분에게 적합합니다.

👉 **[Tandem Robotics (Legacy v1)](https://github.com/wookjin-chung/Tandem_Robotics)**
